关于Demucs音频分离技术

Demucs是由Facebook AI Research开发的先进音频源分离模型，能够将混合音频高质量地分离为不同的音轨组件。

主要特点

  • 基于深度学习的音频源分离
  • 支持多种分离模型，满足不同需求
  • 可分离人声、鼓点、贝斯和其他乐器
  • 支持高级6音轨模型(含钢琴和吉他)
  • 实时显示处理进度
  • 简单易用的Web界面

可用模型说明

htdemucs 默认

特点: Hybrid Transformer Demucs，最新的混合架构模型

适用场景: 高质量分离，平衡了质量和速度

支持音轨: 人声、鼓点、贝斯、其他乐器

htdemucs_ft 高质量

特点: htdemucs的微调版本，更准确但处理速度更慢

适用场景: 需要最高质量分离，对速度要求不高

支持音轨: 人声、鼓点、贝斯、其他乐器

htdemucs_6s 6音轨

特点: 6音轨版本，增加了钢琴和吉他音轨

适用场景: 需要分离钢琴和吉他的场景

支持音轨: 人声、鼓点、贝斯、其他乐器、钢琴、吉他

mdx 平衡

特点: 仅在MusDB HQ上训练的模型

适用场景: 一般用途，性能适中

支持音轨: 人声、鼓点、贝斯、其他乐器

mdx_q 快速

特点: mdx的量化版本，体积更小，处理速度更快

适用场景: 速度敏感场景，对质量要求不高

支持音轨: 人声、鼓点、贝斯、其他乐器

使用建议

  • 对于大多数用户，推荐使用默认的htdemucs模型，它提供了良好的平衡。
  • 如果对分离质量有更高要求，可以尝试htdemucs_ft模型。
  • 如果需要分离钢琴和吉他，请选择htdemucs_6s模型。
  • 处理大型文件可能需要较长时间，尤其是使用高质量模型时。
  • 音频质量较好的文件通常能获得更好的分离效果。

API接口文档

本应用提供REST API接口，支持通过编程方式使用音频分离功能。以下是可用的API端点及其用法。

开发者指南： 以下API接口可用于集成Demucs音频分离功能到您自己的应用程序中。 处理过程是异步的，提交任务后您会获得一个任务ID，可以用它来获取进度更新和最终下载分离结果。 所有响应都采用统一的JSON格式，包含status和data两个主要字段。

1. 获取可用模型

端点: GET /api/models

描述: 返回系统中可用的所有音频分离模型列表

示例响应:

{
  "status": "success",
  "data": {
    "models": [
      {
        "name": "htdemucs",
        "description": "Hybrid Transformer Demucs",
        "stems": ["vocals", "drums", "bass", "other"]
      },
      {
        "name": "htdemucs_ft",
        "description": "Fine-tuned Hybrid Transformer Demucs",
        "stems": ["vocals", "drums", "bass", "other"]
      },
      ...
    ]
  }
}

使用示例 (cURL):

curl -X GET http://localhost:8080/api/models

2. 提交音频处理任务

端点: POST /api/process

描述: 上传音频文件并开始分离处理

请求参数:

  • file - 要处理的音频文件 (必需，multipart/form-data)
  • model - 要使用的模型名称 (可选，默认: htdemucs)
  • stems - 要分离的音轨，逗号分隔 (可选，默认: 全部音轨)

示例响应:

{
  "status": "success",
  "data": {
    "job_id": "85f3da9c-7214-4892-b7bd-1e6e498c5d7b",
    "message": "Audio separation started",
    "status_url": "/api/status/85f3da9c-7214-4892-b7bd-1e6e498c5d7b",
    "progress_url": "/api/progress/85f3da9c-7214-4892-b7bd-1e6e498c5d7b",
    "download_url": "/api/download/85f3da9c-7214-4892-b7bd-1e6e498c5d7b"
  }
}

使用示例 (cURL):

curl -X POST \
  http://localhost:8080/api/process \
  -F "file=@song.mp3" \
  -F "model=htdemucs" \
  -F "stems=vocals,drums"

使用示例 (JavaScript):

const formData = new FormData();
formData.append('file', audioFile); // File对象
formData.append('model', 'htdemucs');
formData.append('stems', 'vocals,drums');

fetch('/api/process', {
  method: 'POST',
  body: formData
})
.then(response => response.json())
.then(data => console.log(data));

3. 获取任务状态

端点: GET /api/status/{job_id}

描述: 获取指定任务的当前状态和进度

示例响应:

{
  "status": "success",
  "data": {
    "job_id": "85f3da9c-7214-4892-b7bd-1e6e498c5d7b",
    "progress": 65,
    "message": "正在处理音频...",
    "status": "processing"
  }
}

使用示例 (cURL):

curl -X GET http://localhost:8080/api/status/85f3da9c-7214-4892-b7bd-1e6e498c5d7b

4. 实时进度更新

端点: GET /api/progress/{job_id}

描述: 通过Server-Sent Events (SSE)接收实时进度更新

返回: text/event-stream 格式的数据流

使用示例 (JavaScript):

const eventSource = new EventSource('/api/progress/85f3da9c-7214-4892-b7bd-1e6e498c5d7b');

eventSource.onmessage = (event) => {
  const progress = JSON.parse(event.data);
  console.log(`进度: ${progress.progress}%, 消息: ${progress.message}`);
  
  // 当任务完成或出错时关闭连接
  if (progress.status === 'completed' || progress.status === 'error') {
    eventSource.close();
  }
};

5. 下载处理结果

端点: GET /api/download/{job_id}

描述: 下载已完成任务的分离音轨（ZIP格式）

返回: application/zip 文件

使用示例 (cURL):

curl -X GET \
  http://localhost:8080/api/download/85f3da9c-7214-4892-b7bd-1e6e498c5d7b \
  --output separated_tracks.zip

使用示例 (JavaScript):

// 使用浏览器直接下载
window.location.href = '/api/download/85f3da9c-7214-4892-b7bd-1e6e498c5d7b';

6. 清理任务文件

端点: DELETE /api/cleanup/{job_id}

描述: 清理指定任务的相关文件，释放服务器存储空间

示例响应:

{
  "status": "success",
  "data": {
    "message": "任务文件已成功清理"
  }
}

使用示例 (cURL):

curl -X DELETE http://localhost:8080/api/cleanup/85f3da9c-7214-4892-b7bd-1e6e498c5d7b
注意: 所有API请求返回的HTTP状态码为200（成功）或4xx/5xx（错误）。响应体中的status字段会指示具体操作是成功还是失败。系统会在一定时间后（默认1小时）自动清理未被下载的文件。

API接口调用关系

典型使用流程

以下是音频分离处理的完整流程和API调用顺序：

  1. 获取可用模型列表 (可选)

    调用 GET /api/models 获取可用的模型列表，用于在界面上展示选项。

  2. 提交音频处理任务 (必需)

    调用 POST /api/process 上传音频文件并获取任务ID (job_id)。

  3. 监控处理进度 (二选一)
    • 方法1: 轮询

      定期调用 GET /api/status/{job_id} 获取最新进度。

      // 示例: 每2秒查询一次进度
const checkStatus = (jobId) => {
  const intervalId = setInterval(async () => {
    const response = await fetch(`/api/status/${jobId}`);
    const data = await response.json();
    
    if (data.status === 'success') {
      const progress = data.data;
      
      // 更新UI显示进度
      updateProgressUI(progress.progress, progress.message);
      
      // 如果任务完成或出错，停止轮询
      if (progress.status === 'completed' || progress.status === 'error') {
        clearInterval(intervalId);
        handleTaskCompletion(progress, jobId);
      }
    }
  }, 2000);
};
    • 方法2: SSE实时更新 (推荐)

      建立 GET /api/progress/{job_id} SSE连接，接收实时进度更新。

      // 示例: 建立SSE连接获取实时进度
const connectToSSE = (jobId) => {
  const eventSource = new EventSource(`/api/progress/${jobId}`);
  
  eventSource.onmessage = (event) => {
    const progress = JSON.parse(event.data);
    
    // 更新UI显示进度
    updateProgressUI(progress.progress, progress.message);
    
    // 如果任务完成或出错，关闭SSE连接
    if (progress.status === 'completed' || progress.status === 'error') {
      eventSource.close();
      handleTaskCompletion(progress, jobId);
    }
  };
  
  eventSource.onerror = () => {
    // 处理连接错误
    console.error('SSE连接错误');
    eventSource.close();
    
    // 可以尝试回退到轮询方式
    checkStatus(jobId);
  };
};
  4. 下载处理结果 (任务完成后)

    任务完成后调用 GET /api/download/{job_id} 下载分离的音轨。

    // 示例: 下载处理结果
const downloadResult = (jobId) => {
  window.location.href = `/api/download/${jobId}`;
  
  // 或者使用fetch并保存文件
  fetch(`/api/download/${jobId}`)
    .then(response => response.blob())
    .then(blob => {
      const url = window.URL.createObjectURL(blob);
      const a = document.createElement('a');
      a.href = url;
      a.download = 'separated_tracks.zip';
      document.body.appendChild(a);
      a.click();
      window.URL.revokeObjectURL(url);
    });
};
  5. 清理资源 (可选)

    下载完成后调用 DELETE /api/cleanup/{job_id} 清理服务器上的临时文件。

    // 示例: 清理任务资源
const cleanupResources = async (jobId) => {
  try {
    const response = await fetch(`/api/cleanup/${jobId}`, {
      method: 'DELETE'
    });
    const data = await response.json();
    console.log('资源清理结果:', data);
  } catch (error) {
    console.error('清理资源出错:', error);
  }
};

错误处理策略

在调用API时，应当实现以下错误处理策略：

  • 网络错误: 检测网络连接问题，必要时重试请求
  • 服务器错误: 适当展示错误信息，提供重试选项
  • SSE连接中断: 如果SSE连接失败，回退到轮询方式获取进度
  • 任务处理错误: 显示详细错误信息，指导用户可能的解决方案

接口依赖关系

各API接口之间的依赖关系：

  • /api/status/{job_id} 和 /api/progress/{job_id} 依赖于先调用 /api/process 获取的job_id
  • /api/download/{job_id} 依赖于任务处理完成（状态为"completed"）
  • /api/cleanup/{job_id} 应在下载完成后调用，以释放服务器资源
