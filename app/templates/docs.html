🚀 快速开始

三个简单步骤即可开始使用：

1
选择音频文件
2
配置参数
3
下载结果
  1. 上传音频文件：支持MP3、WAV、FLAC等常见格式，建议文件大小不超过100MB
  2. 选择处理参数：根据需要选择分离模型、音轨类型、输出格式和音质
  3. 开始处理：点击"开始分离"，系统会实时显示处理进度

💡 新手建议： 初次使用建议选择默认的 htdemucs 模型和 high 质量设置， 可以获得质量和速度的最佳平衡。

🎯 模型选择指南

不同的模型适用于不同的应用场景，选择合适的模型可以显著提升分离效果：

htdemucs 推荐

特点： 平衡质量和速度的最佳选择

适用： 流行音乐、摇滚、电子音乐

处理时间： 中等

输出质量： 高

htdemucs_ft 高质量

特点： 微调版本，质量最佳

适用： 专业制作、高质量要求

处理时间： 较长

输出质量： 最高

htdemucs_6s 特殊

特点： 支持6音轨分离

适用： 包含钢琴和吉他的音乐

处理时间： 较长

输出质量： 高（6轨）

mdx 平衡

特点： 平衡型，适用性广

适用： 各类音乐风格

处理时间： 中等

输出质量： 中高

mdx_q 快速

特点： 快速处理版本

适用： 快速预览、测试用途

处理时间： 最短

输出质量： 中等

🎯 选择建议

  • 日常使用：htdemucs（默认）
  • 专业制作：htdemucs_ft
  • 有钢琴/吉他：htdemucs_6s
  • 快速预览：mdx_q

🎵 音质设置

根据用途选择合适的音质设置，平衡音质、文件大小和处理时间：

质量等级 比特率 采样率 文件大小 适用场景
lossless 无损 N/A 44kHz 最大 专业制作、母带处理
high 推荐 320kbps 44kHz 较大 日常使用、高质量需求
medium 192kbps 44kHz 中等 移动设备、网络分享
low 128kbps 22kHz 最小 快速预览、测试用途

⚠️ 重要提醒

音质设置只对MP3格式有效。WAV和FLAC格式会自动使用最佳质量设置。

📁 输出格式选择

根据用途选择最适合的输出格式：

WAV 无损

优点：

  • 完全无损音质
  • 兼容性最好
  • 适合进一步编辑

缺点：

  • 文件体积大
  • 不支持元数据

适用： 专业制作、音频编辑

MP3 通用

优点：

  • 文件体积小
  • 兼容性极佳
  • 支持元数据

缺点：

  • 有损压缩
  • 不适合多次编辑

适用： 日常使用、网络分享

FLAC 平衡

优点：

  • 无损压缩
  • 文件相对较小
  • 支持元数据

缺点：

  • 兼容性一般
  • 处理时间稍长

适用： 高品质收藏、存档

🎯 格式选择建议

  • 专业用途：选择WAV，获得最佳质量
  • 日常使用：选择MP3 + high质量，平衡质量和大小
  • 高品质收藏：选择FLAC，无损且文件较小
  • 快速分享：选择MP3 + medium质量

🎼 音轨类型说明

了解不同音轨的内容，选择您需要的分离结果：

🎤 Vocals (人声)

包含主唱、和声、说唱等所有人声内容

适用： 卡拉OK制作、人声分析、翻唱

适用： 节拍分析、鼓点替换、混音

🎸 Bass (贝斯)

包含电贝斯、低音提琴等低频乐器

适用： 低频分析、贝斯线学习

🎹 Other (其他)

包含吉他、键盘、弦乐等其他乐器

适用： 伴奏制作、乐器学习

🎹 Piano (钢琴)

专门分离的钢琴音轨（仅htdemucs_6s支持）

适用： 钢琴学习、古典音乐分析

🎸 Guitar (吉他)

专门分离的吉他音轨（仅htdemucs_6s支持）

适用： 吉他学习、摇滚音乐分析

💡 选择建议： 可以同时选择多个音轨类型。如果不确定需要哪些， 建议选择全部基础音轨（人声、鼓点、贝斯、其他），然后根据需要使用特定音轨。

⚙️ 高级使用技巧

批量处理

通过API接口可以实现批量音频处理：

# Python批量处理示例
import requests
import os

def batch_process(file_list, model='htdemucs'):
    results = []
    for file_path in file_list:
        with open(file_path, 'rb') as f:
            files = {'file': f}
            data = {'model': model, 'stems': 'vocals,drums'}
            response = requests.post('http://localhost:8080/api/process', 
                                   files=files, data=data)
            if response.status_code == 200:
                results.append(response.json()['data']['job_id'])
    return results

自定义音质配置

针对不同用途优化音质设置：

  • 广播电台：MP3 + medium质量（节省带宽）
  • 音乐制作：WAV + lossless质量（最佳质量）
  • 移动应用：MP3 + low质量（节省存储）
  • 在线试听：MP3 + high质量（平衡质量和加载速度）

实时进度监控

使用SSE接口监控处理进度：

# JavaScript SSE监控示例
const eventSource = new EventSource('/api/progress/' + jobId);
eventSource.onmessage = (event) => {
    const data = JSON.parse(event.data);
    console.log(`进度: ${data.progress}%`);
    if (data.status === 'completed') {
        eventSource.close();
        downloadResult(jobId);
    }
};

🔧 故障排除

常见问题

❓ 上传失败

可能原因：

  • 文件格式不支持（支持：MP3、WAV、FLAC、OGG、M4A）
  • 文件过大（限制：100MB）
  • 网络连接问题

解决方案：

  • 检查文件格式和大小
  • 尝试压缩音频文件
  • 检查网络连接

❓ 处理速度慢

影响因素：

  • 文件大小和长度
  • 选择的模型（htdemucs_ft最慢，mdx_q最快）
  • 服务器负载

优化建议：

  • 选择较快的模型（mdx_q）
  • 处理较短的音频片段
  • 避免高峰时段

❓ 分离质量不佳

可能原因：

  • 源音频质量较低
  • 音乐风格与模型不匹配
  • 输出质量设置过低

改进方法：

  • 使用高质量源文件
  • 尝试不同模型
  • 提高输出质量设置

💡 最佳实践

🎯 源文件选择

  • 使用尽可能高质量的源文件
  • 避免已经过度压缩的音频
